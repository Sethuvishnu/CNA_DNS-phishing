// 01 — Overview

What is a MITM Attack?

🎯 Project Objective

Demonstrate how an attacker positioned on the same local network can intercept, monitor, and manipulate all network traffic flowing between a victim device and its gateway — using only open-source Python tools and standard Linux utilities.

🔬 What is MITM?

A Man-in-the-Middle attack occurs when an attacker secretly intercepts communication between two parties. The attacker relays and potentially alters communications while both parties believe they're communicating directly with each other.

⚡ ARP Protocol Weakness

ARP (Address Resolution Protocol) has no authentication mechanism. Any device can send fake ARP replies claiming to be any other device. This fundamental design flaw is what makes ARP Spoofing possible on any local network.

🛡️ Why This Matters

Understanding these attacks is critical for network security professionals. By demonstrating exactly how attackers operate, we can build better defenses, implement proper security controls, and educate users about network security risks.

Discovering the Target

The first step in any network attack is reconnaissance — identifying all devices on the network, their IP addresses, and MAC addresses. We used standard Linux networking tools to discover our target device (Phone 1) connected to our hotspot.

BASH — Network Discovery
# Check network interfaces
ip link show
3: wlp4s0: <BROADCAST,MULTICAST,UP,LOWER_UP>
inet 10.42.0.1/24 scope global wlp4s0

# Scan for all connected devices
nmap -sn 10.42.0.0/24
Nmap scan report for victus (10.42.0.1)
Nmap scan report for TARGET (10.42.0.46)

# Verify with ARP table
ip neigh
10.42.0.46 dev wlp4s0 lladdr c2:ec:fe:c4:d1:7e REACHABLE
What we found: Phone 1 at IP 10.42.0.46 with MAC address c2:ec:fe:c4:d1:7e
02
// ARP Cache Poisoning
ARP Spoofing Attack

ARP Spoofing works by sending fake ARP (Address Resolution Protocol) replies to the target device. We tell Phone 1 that our laptop's MAC address is the gateway's MAC address. This causes all of Phone 1's traffic to flow through our laptop instead of going directly to the router.

BASH — ARP Spoofer
sudo python3 arpspoof.py 10.42.0.46 -f

DNS Spoofing Attack

DNS Spoofing takes the attack further — instead of just observing traffic, we actively redirected it. By sending fake DNS responses, we made Phone 1 go to our laptop's web server instead of the real website, while the victim has no idea the redirection is happening.

# Configure DNS spoofing in Bettercap

set dns.spoof.address 10.42.0.1
dns.spoof on

// 05 — Theory

Network Concepts Demonstrated

01
ARP Protocol

Address Resolution Protocol maps IP addresses to MAC addresses on a local network. Since ARP has no authentication, any device can send fake ARP replies — poisoning the ARP cache of other devices.

02
DNS Protocol

Domain Name System translates human-readable domains (google.com) to IP addresses. By intercepting and forging DNS responses, an attacker can redirect victims to any IP address they control.

03
OSI Layer 2 & 3

ARP operates at Layer 2 (Data Link) using MAC addresses. IP routing operates at Layer 3 (Network). Our attack exploits both layers — poisoning MAC-IP mappings to intercept Layer 3 traffic.

04
IPv4 Forwarding

When enabled, allows a Linux system to forward packets between interfaces — acting as a router. Essential for our MITM attack so the victim's internet continues working while we intercept traffic.

05
HTTP vs HTTPS

HTTP traffic is completely readable — passwords, messages, images all visible in Wireshark. HTTPS encrypts content but DNS metadata still leaks which sites are visited, which apps are active.

06
Packet Analysis

By analyzing packet metadata — source IP, destination IP, DNS queries, SNI fields — we could identify exactly which apps and services the target was using, even with encrypted HTTPS traffic.

// 07 — Protection

Defense Mechanisms

🛡️ Against ARP Spoofing

Static ARP Entries — Manually configure ARP table so it cannot be poisoned by fake replies.

sudo arp -s 10.42.0.1 10:68:38:26:5d:63
# Locks gateway MAC — ignores fake ARP

VPN — Encrypts all traffic at the network layer, making ARP spoofing interception useless even if MITM position is achieved.

🔒 Against DNS Spoofing

DNSSEC — DNS Security Extensions cryptographically sign DNS records, making fake DNS replies detectable and rejected.

DNS over HTTPS (DoH) — Encrypts DNS queries so they cannot be intercepted or spoofed by a network-level attacker.

# Use encrypted DNS
DNS Server: 1.1.1.1 (Cloudflare DoH)

🌐 Against Traffic Interception

Always Use HTTPS — Look for the padlock icon. HTTPS encrypts the content of all communications, even if DNS metadata leaks.

HSTS — HTTP Strict Transport Security forces browsers to always use HTTPS for specific domains, preventing SSL stripping attacks.

📱 Why Some Apps Are Safe

Certificate Pinning — Apps like WhatsApp and banking apps hardcode their server certificates. Any MITM certificate is rejected immediately.

End-to-End Encryption — WhatsApp uses E2EE where only sender and receiver have the decryption keys. Even the servers cannot read messages.

💡 Key Security Lessons

Never trust public WiFi
Any network you don't control could have a MITM attacker monitoring your traffic.
Use a VPN
VPN encrypts all traffic from your device, making MITM attacks practically useless.
Check for HTTPS
Always verify the padlock icon before entering any sensitive information.
DNS leaks metadata
Even HTTPS sites reveal which domains you visit through DNS — use DNS over HTTPS.
